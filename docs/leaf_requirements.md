# テキストエディター「Leaf」要件定義

## 技術系
* アプリケーションはPWAとして実装
* アプリケーションはWebAssemblyで実装（言語はRust）
  * DOMの操作やブラウザ機能はJavaScriptライブラリを呼び出す形で使用
* フロントエンドとバックエンドの切り分けは「フロントエンドはバックエンドへ送るデータ収集と表示のみに徹し、ビジネスロジックはバックエンドにしか記述しない。バックエンドは外部からのデータ収集と、フロントエンドとのやりとりを行うREST API実装。」
* フロントエンドの装飾はTailwindを使用
* バックエンドAPIは、Node.js＋Expressを使用
* バックエンドを使用しないで実装出来るのであれば、フロントエンドだけで完結させる

## エディタ系
* アプリ起動時に全画面をopacity:0.8の黒オーバーレイで覆い、画面中央にローディングインジケータを表示
* シートデータの競合チェックなどが終了しても、オーバーレイは最低でも1秒間表示
* 非同期処理などがすべて終了しテキスト編集が可能な状態になり次第オーバーレイを0.3秒でフェードアウト
* テキスト編集は「vim」モードのオン／オフが出来る（デフォルトはオン）
  * テキスト編集には「Ace Editor」ライブラリを使用
  * vimキーバインドのON/OFF情報はブラウザに保存する
  * vimキーバインドONの場合、ショートカットとしてMacはCmdキー、WinはWinキーのイベントを奪う
  * Cmd/Winキーのショートカットは以下
    * Cmd/Win+t → 新規シートを開く
    * Cmd/Win+f → テキスト検索
    * Cms/Win+Shift+f → テキスト正規表現検索
* Ctrlキーの動作は、VimキーバインドがONの時は、Vimキーバインドに準拠
  * OFFの場合は、テキストエリアの動作に準拠
* シート画面最上段にボタンが並んだボタンバー、最下段にステータスバーを配置する
  * ボタンバーには左から「新規シート作成」「フロッピーディスクアイコン（保存）」「vimモードON/OFF」を配置
  * ステータスバーには、現在のネットワーク状況を表示する
  * ネットワークの接続状況は定期的にチェックし、接続されていなかった場合は赤色で「Network unreachable」、接続されていたら緑色で「Network connected」とステータスバー最右に表示する
* 開けるシートはひとつだけ

## ブラウザストレージ系
* 設定系のデータはlocalStorageに保存、データ系はIndexedDBを使用
  * IndexedDBは、DBの構造が変更された場合にバージョン番号を上げる
* IndexedDBには現在開かれている、Googleドライブに保存されているシートの情報を、オブジェクトとして保持
  * 各シートオブジェクトにはシート情報が入っている
  * シート情報は以下が入っている
    * カテゴリー（デフォルト：NO_CATEGORY）
    * ファイルパス（マイドライブ/ApplicationSupport/LeafData/[カテゴリー]/[GUIDファイル名]）

## ファイル管理系
* アプリケーション起動直後にIndexedDB上に開かれたシートデータが無かった場合は、なにも書かれていないまっさらなシートを開く
  * IndexedDB上に開かれたシートデータがあった場合は、開かれているシートを開いた状態で復元する
  * シートのテキストが変更されるまでは、Googleドライブには保存しない
  * シートのテキストが変更された時は、ユニークなGUIDファイル名を生成し「マイドライブ/ApplicationSupport/LeafData/[カテゴリー]/[GUIDファイル名]」として保存する
  * 新規シートのカテゴリーは「NO_CATEGORY」
  * ファイルを開くアイコンがクリックされるか、Cmd/Win+oが押されるとファイル選択ダイアログが開く
    * ダイアログのサイズは、ブラウザサイズの縦横ともに80％のサイズ
    * デフォルトでは「[My Drive]/ApplicationSupport」配下のディレクトリ名一覧（カテゴリ一覧）選択ダイアログが開く
    * カテゴリ一覧は縦に並んで表示される
    * カテゴリが選択されると、カテゴリ名のディレクトリ内にあるファイル選択ダイアログになる
    * ファイル選択ダイアログではファイル名を表示して選択するのではなく、ダイアログの縦サイズを5等分して配置したボタンに、ファイルの内容を表示したボタンを選択する
    * カテゴリ一覧、ファイル一覧、ともにカーソル上下で選択を移動出来る
    * エンターキーで選択
    * 選択されたファイルを表示

## ファイル選択ダイアログ系
* ダイアログのレイアウトは、最上部にタイトルバー「File select」
* タイトルバーの下にボタンバー
  * 上部ボタンバーの一番右に、更新アイコンのボタン
* 一番下にもボタンバー
  * 下部ボタンバーには右から「OK」「Cancel」
* 上部・下部ボタンバーの間を左30％：右70％で分割
* 左30％はカテゴリーバーとして、カテゴリ一覧（ディレクトリ一覧）を縦に並べて表示
* カテゴリはカテゴリーバーエリアの縦サイズを10分割したサイズの95％を縦サイズにしたボタンに、カテゴリ名を表示して配置
* カテゴリが10個以上ある場合はスクロール
* 右70%はファイル一覧
* アプリケーションが開かれた時に、カテゴリー一覧（ディレクトリ一覧）を取得し、IndexedDBに保存
* 上部ボタンバーの更新ボタンがクリックされた場合にIndexedDBのカテゴリ一覧を更新する
* カテゴリーバーのカテゴリーをクリック／選択されると、ファイル一覧にカテゴリ（ディレクトリ内）のファイル一覧を縦に並べて表示する
* ファイルはファイル一覧エリアを5分割したサイズの95％を縦サイズにしたボタンに、ファイルの先頭からのテキストを表示
* ファイルが5個以上ある場合はスクロールする
* カテゴリーバーとファイル一覧エリアはタブキーでフォーカスを行き来する
* カテゴリー一覧、ファイル一覧ともにカーソルキーで選択位置を移動出来る
* 選択されているカテゴリ／ファイルボタンは、lime色の枠が付く（枠の太さは2.0px）
* ダイアログ内に表示するボタンは全て、6pxで角を丸め、ドロップシャドウさせる
* ファイルボタンはクリックするだけではファイル読み込みにはならず、クリックで選択、「OK」ボタンをクリックするかエンターキーが押された時に読み込まれる

## 自動保存系
* Googleドライブに保存されたファイルは自動保存される
  * 最後に編集されてから1秒間、なにも編集作業が行われなかった場合に非同期で自動保存される
  * 自動保存される場所は、「マイドライブ/ApplicationSupport/LeafData/[カテゴリー]/[GUIDファイル名]」
* テキストファイルを読み込んで開いた時に、文字コードがBOM付きUTF-8ではなかった場合はBOM付きUTF-8に変換するかのダイアログを出す
  * 「OK」がクリックされた時は、以降そのファイルはBOM付きUTF-8として扱う（自動保存もBOM付きUTF-8で上書き保存する）
* ネットワークが切断されている場合は、現在開いているシートの自動保存をIndexedDBにテンポラリ保存する
* テンポラリ保存シートには、保存した時刻データを持たせる（ネットワークが復帰した場合にGoogleドライブ上のファイルと新旧比較するため）
* テンポラリへの自動保存も非同期で行う
* ユーザーには、右下のネットワーク状態表示以外での通知は行わない
* ネットワーク接続が復帰した場合には、テンポラリ保存していたシートデータをGoogleドライブへ保存し、テンポラリシートは削除
* アプリが開かれた時にIndexedDBにテンポラリ保存のシートがあった場合は、そちらをエディタ上へ復元
* テンポラリ保存シートを復元した後にネットワークが接続状態だった場合は、Googleドライブにあるファイルの時刻データとテンポラリ保存シートの保存時刻データを比較し、Googleドライブ上のファイルの方が新しかった場合はどちらを残すかをユーザーに訊くダイアログを表示

## アカウント系
* 使用するにはGmailアカウントでログインが必要
  * ログイン時に取得する情報はメールアドレスと、GoogleドライブへのRead/Write権限
  * Gmailログイン後に「マイドキュメント/ApplicationSupport/LeadData」というディレクトリが無ければ作成する
  * 「マイドキュメント/ApplicationSupport/LeadData/NO_CATEGORY」というディレクトリも無ければ作成する

# バージョン番号表示系
* package.jsonのバージョン番号を使用してバージョン管理を行う
* 最初は、ver0.1.0から始める
* バグ修正や細かい修正の場合は、バージョン番号を「0.0.1」上げる
* 機能追加や、仕様変更を行った場合はバージョン番号を「0.1.0」上げる
* バージョンをフッターの一番左に表示する

## アプリケーション起動フロー
:::mermaid
graph TD
    A[アプリケーション起動] --> B[/Gmailログインしているか/];
    B -- している --> G[新規シートを開く];
    B -- していない --> C[Googleログイン画面を開く];
    C --> D[ログインする];
    D --> E[/ログインエラー/];
    E -- エラーの場合 --> F[Gmailのエラー表示];
    E -- 成功の場合 --> G;
:::

## 新規ファイル生成フロー
:::mermaid
graph TD
    A[新規ファイル作成] --> B[/テキストデータ更新?/]
    B -- されていない --> C[待機]
    C --> B
    B -- された --> D[GUIDファイル名生成]
    D --> E["IndexedDBに保存<br/>(GUID, カテゴリー, テキストデータ)"]
    E --> F["Googleドライブに保存<br/>(マイドライブ/ApplicationSupport/LeafData/NO_CATEGORY)"]
:::



# 仕様修正
* メニューバーの高さを、現在の80％にする
* ファイル選択ダイアログのカテゴリ一覧を10分割から15分割にする
* アプリを開いた時、IndexedDBに編集中のシートがあった場合、Googleドライブ上のテキストとIndexedDB上のテキストを比較し、差異があった場合はどうするか選択するダイアログ（実装済み）を表示

# 仕様追加
* メニューバーに「ー」「＋」のボタンを配置（フォントサイズ変更）
* Opt/Alt+マイナス、Opt/Alt+イコールのショートカットが押されてもフォントサイズを変更

# バグ
* 新規シートを保存した後に、再び新規シートを作って保存しようとしても保存されない（ログになにも出ない）
* [解決]アプリを開いた直後の新規シートに文字を書くと、最初の文字追加がキャンセルされる（Aceの問題かも？）
* [解決]アプリが開いた直後にGoogleドライブからシートを読み込むと、エディタ部が新規シートのままになっている
  * IndexedDBは読み込んだシート情報になっている
  * リロードすると読み込んだシートの編集状態になっている
* [解決]シートをGoogleドライブから開いた後に、Opt/Alt+t を押して新規シートにしても行番号が赤くならない
